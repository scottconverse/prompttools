2. What prompttools Is

prompttools is a suite of free, open-source developer tools for managing LLM prompt files:

  • promptlint — Static analysis tool that checks prompts for token bloat, security vulnerabilities, hallucination risk, structural problems, variable issues, and pipeline integrity.
  • promptfmt — Auto-formatter that normalizes whitespace, delimiters, variable syntax, line wrapping, and YAML/JSON structure in prompt files.
  • prompttest — Test framework that runs YAML-defined assertions against prompt files (content checks, token limits, cost budgets, format validation).
  • promptcost — Cost estimator that calculates token costs across models, projects volume-based spending, and enforces budget limits.
  • promptdiff — Semantic diff tool that compares prompt files and detects message-level changes, variable impact, token deltas, and breaking changes for CI/CD gating.
  • promptvault — Version control and registry for prompt assets with semantic versioning, dependency resolution, lockfile management, and a searchable catalog.
  • prompttools-core — Shared foundation library providing prompt parsing, tokenization, model profiles, configuration, and caching.

These are development tools — they do not execute prompts, connect to AI services, or send data to external servers. promptfmt and promptlint's --fix flag modify files only when explicitly requested.

5. Tool-Specific Risks

5.1 Lint Results Are Advisory

promptlint identifies potential issues based on pattern matching and heuristic analysis. It does not understand the semantic intent of your prompts. A clean lint pass does not mean your prompts are safe, effective, or free of problems. Conversely, lint warnings may flag patterns that are intentional and correct in your context.

5.2 Security Analysis Limitations

The security rules (PL060-063) detect common patterns for PII, API keys, and injection vectors. They are not a comprehensive security audit. Prompts that pass security linting may still contain sensitive information, be vulnerable to injection attacks, or have other security issues that pattern-based analysis cannot detect.

5.3 Formatting Risk

promptfmt modifies prompt files to normalize formatting. While it includes semantic equivalence checking, automated file modification carries risk. Always review changes and use version control.

5.4 Cost Estimation Limitations

promptcost estimates are based on published model pricing and heuristic output token estimation. Actual API costs may differ due to pricing changes, actual output length variation, and tokenizer approximations. Cost estimates should not be used as the sole basis for financial decisions.

5.5 Test Framework Limitations

prompttest validates prompts against user-defined assertions. Passing all tests does not guarantee prompt correctness, safety, or fitness for purpose. Test coverage is only as good as the assertions you write.

5.6 Diff Analysis Limitations

promptdiff classifies changes as breaking or non-breaking based on heuristic analysis. It may not detect all breaking changes in all contexts. A change classified as non-breaking may still affect prompt behavior in ways that pattern-based analysis cannot predict.

5.7 Registry and Versioning Limitations

promptvault manages prompt packages in a local file-system registry. Lockfile integrity checking uses content hashing but does not guarantee that prompt packages are safe, correct, or free of issues. Dependency resolution follows semver conventions but cannot verify semantic compatibility between prompt versions.

5.8 Auto-Fix Risk

The --fix flag in promptlint automatically modifies prompt files. While fixes target specific, well-defined issues (trailing whitespace, formatting), any automated file modification carries risk. Always review changes after using --fix and use version control.
